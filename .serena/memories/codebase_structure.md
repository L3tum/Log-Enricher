# Codebase structure
- Root files: `main.go` executable entrypoint; `main_test.go` and `main_e2e_integration_test.go` root tests; `README.md` usage/config guide; `AGENTS.md` contributor guardrails; `Dockerfile` container build.
- Top-level directories: `docs/` for documentation; `internal/` for implementation packages; `.github/` and `.devcontainer/` for automation/dev environment.
- Important `internal/` packages:
  - `internal/config`: environment-driven configuration loading.
  - `internal/tailer`: file discovery, watcher management, per-file tailing.
  - `internal/processor`: line-to-entry processing and backend send path.
  - `internal/pipeline`: ordered stage manager and enrichment/filter/parser stages.
  - `internal/backends`: file and Loki backends.
  - `internal/promtailhttp`: Promtail-compatible HTTP receiver.
  - `internal/state`: persisted state for file identity/offset/cache tracking.
  - `internal/models`: canonical `models.LogEntry` data model.
  - `internal/logging`: custom slog handler wiring logs to stdout and backend.
  - Supporting packages include `internal/network`, `internal/cache`, and `internal/bufferpool`.
- Architectural guardrail: do not reintroduce removed legacy APIs such as `backends.Manager`, `cfg.Backends`, `GetLogEntry`/`PutLogEntry`, or `internal/parser`; migrate callers/tests or clean up legacy code in a dedicated change instead.
