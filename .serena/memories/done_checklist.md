# What to do when a task is completed
- Run the required verification from `AGENTS.md` before considering work complete:
  - `go test ./...`
  - `go test -race ./...`
  - `go test -count=1 ./...`
  - `staticcheck ./...`
- If full verification is too slow during development, at least run tests for touched packages, then still run `go test ./...` before merge.
- If you changed exported interfaces or cross-package APIs, update every caller and all affected tests in the same change.
- After interface changes, verify stale symbol names/signatures are gone with `rg "OldSymbolName|old_signature_fragment"` (adapt pattern to the actual change).
- If file identity/tailer/state logic changed, ensure tests cover unchanged file, rotation, truncation, and missing file cases.
- If work affects network-bound code or goroutines, confirm context propagation and cancellation behavior are preserved.
